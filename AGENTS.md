# 项目级 Codex 指令

## Skill 优先级

- 处理需求梳理、PRD、业务流程、页面改造、交互原型或验收设计时，必须优先使用本项目的 `.agents/skills/requirement-clarifier/SKILL.md`。
- 如果环境中存在同名全局 Skill，本项目版本是当前仓库的权威版本；只有项目版本缺失或无法读取时才允许使用全局版本，并向用户说明。

## Windows 编码门禁

- 本项目 Markdown 文件使用 UTF-8。通过 Windows PowerShell 读取时必须显式指定 `-Encoding UTF8`。
- 在 Windows 上运行 Python 校验脚本前设置 `PYTHONUTF8=1`，避免脚本按系统 GBK 默认编码读取 UTF-8 文件。
- 如果读取结果出现乱码、Unicode 替换字符 `U+FFFD`、异常汉字组合或不可理解的指令，立即停止任务动作并以 UTF-8 重新读取完整文件；禁止依据乱码内容继续分析或输出。

## 需求交付门禁

- 用户逐轮确认规则时，只更新确认结论；不得把局部规则答复表述为完整需求交付。
- 满足停止追问条件后，必须按照 Skill 选定模式生成正式方案，并执行完成前自检。
- 只有正式方案覆盖目标、角色权限、主流程、状态、规则、页面、关键交互、异常和可测试验收后，才能表述为“需求已对齐”或“方案已完成”。
