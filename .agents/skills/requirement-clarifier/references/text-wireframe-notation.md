## 1. 原则

文字原型用于确认信息结构、任务路径、状态和交互逻辑，不追求视觉稿的像素精度。

遵守：

- 只画支持核心闭环和高风险分支的关键页面或节点。
- 每个原型标明名称、角色、入口和状态。
- 原型中的按钮、字段、状态名称与需求正文一致。
- 不使用真实数据时，使用占位值并注明“示例”。
- 对不同角色或状态的差异分别画变体，不在一个框中混杂所有可能。
- 原型后必须附交互说明，不能只给图形。
- 中文全角字符可能造成对齐差异，以结构和语义清楚为优先。

## 2. 基础符号

```text
[按钮]                 可执行操作
[输入内容________]     输入框
[请选择          ▼]    选择器
( ) 单选  [ ] 多选     选择控件
< 返回                 返回或上一级
⋯                      更多操作
! 提示                  警告或错误
✓                      成功
?                      待确认或帮助
{条件？}                判断节点
→                      跳转或状态变化
│                      分隔或泳道
```

状态标记：

```text
[默认]
[加载中]
[空]
[错误]
[无权限]
[已禁用]
[只读]
```

## 3. 移动端与 h5

### 页面框架

```text
┌────────────────────────────┐
│ < 返回       页面标题       │
├────────────────────────────┤
│ 状态提示、步骤或摘要         │
├────────────────────────────┤
│ 字段名称 *                  │
│ [请输入__________________]  │
│ 辅助说明或错误提示           │
│                            │
│ 字段名称                    │
│ [请选择                 ▼]  │
│                            │
│ 上传材料                    │
│ [ + 上传 ]                 │
├────────────────────────────┤
│ [次要操作]        [主要操作]│
└────────────────────────────┘
```

### 列表页面

```text
┌────────────────────────────┐
│ < 返回       记录列表       │
├────────────────────────────┤
│ [搜索__________________ 🔍] │
│ [状态▼] [类型▼] [日期▼]     │
├────────────────────────────┤
│ 标题 A             [处理中] │
│ 摘要信息                    │
│ 2026-07-10              >   │
├────────────────────────────┤
│ 标题 B             [已完成] │
│ 摘要信息                    │
│ 2026-07-09              >   │
├────────────────────────────┤
│             [ + 新建 ]      │
└────────────────────────────┘
```

### 移动端交互重点

说明：

- 滚动时底部按钮是否固定。
- 键盘弹出后的页面行为。
- 日期、上传、定位、扫码等系统能力。
- 返回时未保存内容处理。
- 网络中断与重复点击防护。
- 通知点击后的深链页面。

## 4. web 与管理后台

```text

│ logo │ 一级导航                    用户 / 组织 / 帮助      │

│      │       [查询] [重置] [更多筛选] [导出]              │
│      ├───────────────────────────────────────────────────┤
│      │ [ ] 名称 │ 类型 │ 状态 │ 更新时间 │ 负责人 │ 操作  │

```

### 详情页面

```text
┌──────────────────────────────────────────────────────────┐
│ < 返回列表  记录编号 #0001     [状态]   [次要] [主要操作] │
├──────────────────────────────────────────────────────────┤
│ 基本信息                    │ 当前处理信息                │
│ 字段：值                    │ 当前节点：……                │
│ 字段：值                    │ 当前负责人：……              │
├────────────────────────────┴─────────────────────────────┤
│ [详情] [附件] [操作记录] [关联数据]                       │
├──────────────────────────────────────────────────────────┤
│ 当前标签页内容                                             │
└──────────────────────────────────────────────────────────┘
```

### 后台交互重点

说明：

- 默认筛选和默认排序。
- 查询是否实时、点击触发或防抖。
- 多选、跨页选择和批量部分成功。
- 列配置、固定列、横向滚动和数据密度。
- 导出范围、权限、水印和异步任务。
- 详情抽屉与独立详情页的使用边界。

## 5. 弹窗、抽屉与状态变体

### 二次确认弹窗

```text
┌────────────────────────────┐
│ 确认执行高风险操作？    ×   │
├────────────────────────────┤
│ 此操作将产生：……           │
│ 影响对象：……               │
│ 是否可撤销：否              │
│                            │
│ [ ] 我已了解影响            │
├────────────────────────────┤
│ [取消]              [确认]  │
└────────────────────────────┘
```

### 空状态

```text
┌────────────────────────────┐
│                            │
│          暂无记录           │
│  完成首次创建后将在这里显示  │
│          [立即创建]         │
│                            │
└────────────────────────────┘
```

### 错误状态

```text
┌────────────────────────────┐
│ ! 加载失败                  │
│ 暂时无法获取数据            │
│ [重试]  [联系支持]          │
└────────────────────────────┘
```

对一个关键页面至少描述默认、加载、空、错误、无权限和提交中状态中适用的部分。

## 6. 流程图与状态图

### 业务流程

```text
[触发事件]
    ↓
[角色发起]
    ↓
[系统校验]
    ↓
{是否通过？}
  ├─ 否 → [展示原因] → [修改 / 终止]
  └─ 是 → [进入处理]
               ↓
          {处理结果？}
          ├─ 成功 → [通知] → [归档]
          └─ 失败 → [重试 / 人工补偿]
```

### 状态图

```text
[草稿] --提交--> [待处理]
[待处理] --通过--> [已完成]
[待处理] --退回--> [待修改] --重新提交--> [待处理]
[待处理] --撤回--> [已撤回]
[待处理] --超时--> [待升级] --人工处理--> [已完成 / 已关闭]
```

每条状态边必须对应一个触发事件和允许执行该事件的角色。

## 7. 角色泳道

```text
时间 ↓

申请人            系统                 审批人              管理员
  │                 │                    │                   │
  │ 填写并提交 ────>│                    │                   │
  │                 │ 校验并创建任务 ──>│                   │
  │                 │                    │ 查看并处理         │
  │                 │<──── 审批结果 ────│                   │
  │<── 状态与通知 ──│                    │                   │
  │                 │                    │                   │
  │                 │── 超时告警 ──────────────────────────>│
  │                 │                    │                   │ 人工干预
```

泳道后补充每个交接点的输入、输出、时限和失败责任人。

## 8. 服务蓝图与线下触点

```text
阶段：          预约              到场              服务中             完成
────────────────────────────────────────────────────────────────────
用户动作：      选择时段          出示凭证          接受服务           确认结果
可见证据：      确认页/短信        二维码/号码牌       设备/物料           回执/评价入口
前台动作：      确认预约          核验并引导          执行并解释          交付并告知
──────────────── 可见线 ────────────────────────────────────────────
后台动作：      锁定容量          调取信息            记录进度            结算/归档
支持系统：      预约系统          身份/核验系统        工单/库存系统        消息/报表系统
异常处理：      满额候补          凭证无效            设备故障            结果争议
责任人：        运营              现场负责人          服务主管            客服/管理员
```

线下原型还应说明场地、人员、设备、物料和安全应急中适用的部分。

## 10. ai 交互

### 对话或任务界面

```text
┌────────────────────────────────────────┐
│ 智能助手     知识范围：项目资料   [设置]│
├────────────────────────────────────────┤
│ 用户：请生成……                         │
│                                        │
│ 助手：[生成中……]                       │
│                                        │
│ 助手：结果正文                         │
│ 来源：[1] [2]   置信提示：需要人工确认  │
│ [复制] [编辑] [重试] [反馈] [确认执行] │
├────────────────────────────────────────┤
│ [输入问题____________________] [发送]  │
└────────────────────────────────────────┘
```

### 低置信与高风险动作

```text
[模型结果]
    ↓
{置信度或证据是否足够？}
  ├─ 否 → [追问 / 拒答 / 转人工]
  └─ 是 → {是否产生高风险副作用？}
             ├─ 是 → [展示影响] → [人工确认] → [执行]
             └─ 否 → [直接展示或保存草稿]
```

说明引用来源、上下文范围、人工编辑、撤销、反馈和失败降级。

## 12. 规则决策表

```text
| 条件 A | 条件 B | 条件 C | 决策结果 | 后续流程 |
|---|---|---|---|---|
| 是 | <= 3 | 有效 | 允许 | 一级审批 |
| 是 | > 3  | 有效 | 允许 | 两级审批 |
| 任意 | 任意 | 无效 | 拒绝 | 提示并终止 |
```

规则较多时补充优先级：先检查禁止条件，再检查资格，再计算结果，最后决定流程。

## 13. 交互注释

每个关键原型后使用统一注释：

```text
交互说明
- 角色：……
- 入口：……
- 默认状态：……
- 主要操作：……
- 启用或禁用条件：……
- 校验时机与规则：……
- 成功反馈：……
- 失败反馈与恢复：……
- 跳转或下一触点：……
- 返回与内容保留：……
- 权限差异：……
- 数据刷新或同步：……
- 通知、日志与埋点：……
```

只写适用项，但成功、失败、跳转和状态变化不得省略。

## 14. 输出选择规则

- app、h5、小程序：至少输出核心表单或任务页、列表或入口页、详情或结果页。
- web 管理后台：至少输出列表或工作台、详情或处理页、一个高风险弹窗。
- 审批或协作：输出主流程泳道和一个处理节点页面或表单。
- 运营活动：输出用户参与路径、活动页面或触点、运营后台或奖励状态。
- 数据产品：输出看板布局、钻取或筛选逻辑、数据异常状态。
- ai 产品：输出输入与结果界面、人工确认、低置信或失败路径。
- 线下服务：输出服务蓝图、现场核验或交付节点、异常升级。
- api 或纯技术能力：输出调用序列、状态查询和错误补偿，不强行画页面。
- 规则制度：输出决策表、状态机和申诉或例外流程，不强行画页面。

多个形式同时适用时，优先展示端到端主线，再补充最关键的界面或节点，不重复画同一逻辑。
