# 最终输出模板

## 使用规则

1. 根据快速、标准、深度模式选择输出内容，不得机械输出全部章节。
2. 已确认、合理假设、待确认、P0 待确认和不在本期必须明确区分。
3. 用户提供已有系统时增加 AS-IS、TO-BE、GAP、信息冲突和变更影响；全新产品可省略 AS-IS/GAP。
4. 只输出当前需求适用的异常、非功能和实施项，不凭空编造指标、接口或基线。
5. 先识别交付意图：需求方案默认输出；原型说明在用户要求页面、交互或原型时输出；Codex 原型提示词只在用户明确要求提示词或实现指令时输出。
6. 只有目标、角色、主流程、页面范围和关键交互明确且无 P0 时，才生成 Codex 原型提示词。
7. 除非用户明确要求实现 Demo，否则本模板只生成文档。
8. 标准模式使用稳定编号，并在规则、页面、交互和验收中填写关联编号；深度模式再增加完整追踪矩阵。
