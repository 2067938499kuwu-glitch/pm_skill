# V2 真实场景测试用例

## TC-01 一句话模糊需求

- 测试编号：TC-01
- 测试输入：“想做一个员工绩效页面。”
- 预期模式：默认按标准模式评估；若最终明确为低风险的个人已发布绩效查看页，可记录理由降级为快速；涉及评分、审批或统计口径时保持标准或升级深度。
- 预期行为：提取“绩效页面”，一次合并询问使用者、核心任务和数据性质。
- 预期追问数量：1 轮，2—3 问。
- 是否存在 P0：是，使用角色和最终任务未明确。
- 停止追问条件：目标、角色、主流程、权限、页面范围和验收方向明确。
- 不应发生的行为：直接输出完整深度 PRD；先问图表颜色。
- 预期输出：与复杂度匹配的需求与原型说明。

## TC-02 用户提供完整 PRD

- 测试编号：TC-02
- 测试输入：包含目标、角色、流程、状态、页面和验收的完整 PRD。
- 预期模式：标准或深度，按 PRD 复杂度。
- 预期行为：直接提取九步所需信息，只检查冲突和真正缺口。
- 预期追问数量：0—1 轮。
- 是否存在 P0：仅在 PRD 存在根本性冲突时存在。
- 停止追问条件：九步内部检查通过且无 P0。
- 不应发生的行为：把 PRD 已回答内容拆成九轮重问。
- 预期输出：缺口、冲突、方案和可测试验收。

## TC-03 用户连续回复“确认，下一步”

- 测试编号：TC-03
- 测试输入：对每轮建议回复“确认，下一步”。
- 预期模式：保持当前模式。
- 预期行为：记录建议为已确认，自动跨过已满足步骤。
- 预期追问数量：只追问新的 P0；不固定九轮。
- 是否存在 P0：按具体需求。
- 停止追问条件：闭环条件满足且无 P0。
- 不应发生的行为：重复展示完整历史；问是否现在输出。
- 预期输出：直接进入匹配深度的正式方案。

## TC-04 用户回复“不知道”

- 测试编号：TC-04
- 测试输入：对撤回规则回复“不知道”。
- 预期模式：当前模式。
- 预期行为：提供 2—4 个选项、影响和可逆推荐。
- 预期追问数量：1 个决策问题。
- 是否存在 P0：否，除非撤回会决定核心状态闭环。
- 停止追问条件：选择或接受合理假设后继续。
- 不应发生的行为：因用户不知道而停止；将所有选项都设为 P0。
- 预期输出：明确标记的合理假设及影响。

## TC-05 用户要求立即停止追问

- 测试编号：TC-05
- 测试输入：“停止追问，按当前信息输出。”
- 预期模式：当前模式的不完整输出。
- 预期行为：立即停止，列出 P0、假设、假设失效影响和无法承诺。
- 预期追问数量：0。
- 是否存在 P0：如实保留。
- 停止追问条件：用户指令本身。
- 不应发生的行为：继续问任何澄清问题；伪装无 P0。
- 预期输出：带缺口和风险的当前最佳方案。

## TC-06 存在 P0 但用户拒绝回答

- 测试编号：TC-06
- 测试输入：用户拒绝说明谁可审批，但要求继续输出。
- 预期模式：标准/深度草案。
- 预期行为：标记【P0待确认】，给出保守示意但不承诺权限方案。
- 预期追问数量：0，用户已拒绝。
- 是否存在 P0：是，关键权限未明确。
- 停止追问条件：用户主动停止。
- 不应发生的行为：生成可直接实现的权限代码；自行选择最方便实现的角色。
- 预期输出：带阻塞说明的概念级方案。

## TC-07 简单页面优化

- 测试编号：TC-07
- 测试输入：“已有列表的空状态没有操作入口，请加一个新建按钮的原型说明。”
- 预期模式：快速。
- 预期行为：复用现有页面，确认新建权限和跳转。
- 预期追问数量：0—1 轮，最多 3 问。
- 是否存在 P0：通常否；如新建权限完全未知且有越权风险则是。
- 停止追问条件：入口、权限、跳转和反馈明确。
- 不应发生的行为：输出接口、迁移、埋点和回滚全套深度章节。
- 预期输出：简版页面和 3—5 个关键交互说明。

## TC-08 多角色复杂审批

- 测试编号：TC-08
- 测试输入：员工、主管、财务和总经理按金额分支审批。
- 预期模式：深度。
- 预期行为：确认阈值、审批顺序、状态、权限、日志和最终生效。
- 预期追问数量：按 P0 合并为 1—3 轮，不固定九轮。
- 是否存在 P0：是，阈值、主流程结果和关键权限。
- 停止追问条件：审批链、状态机、权限和验收方向闭环。
- 不应发生的行为：先设计按钮位置；遗漏驳回、撤回和日志。
- 预期输出：规则、状态、审计、风险和追踪矩阵。

## TC-09 数据看板和统计口径

- 测试编号：TC-09
- 测试输入：销售转化看板，按部门查看线索到成交转化率。
- 预期模式：深度。
- 预期行为：优先确认数据源、分子/分母、去重、时间归属、部门范围和更新延迟。
- 预期追问数量：1—2 轮高相关问题。
- 是否存在 P0：是，统计口径会导致结果失真。
- 停止追问条件：口径、数据源、权限和验收方向明确。
- 不应发生的行为：先问图表样式；虚构基线和目标值。
- 预期输出：指标字典、权限、页面、数据延迟和口径验收。

## TC-10 已有页面或代码改造

- 测试编号：TC-10
- 测试输入：提供现有列表截图和代码，要求新增批量分配。
- 预期模式：标准。
- 预期行为：先输出 AS-IS/TO-BE/GAP 和保留/新增/修改/废弃。
- 预期追问数量：只问代码不能证明的目标规则和 P0。
- 是否存在 P0：如批量权限或覆盖规则存在根本差异，则是。
- 停止追问条件：GAP、权限、主流程和失败恢复明确。
- 不应发生的行为：把代码当作正确目标；未获授权修改代码。
- 预期输出：已有系统改造方案和变更影响。

## TC-11 只生成 Codex 提示词

- 测试编号：TC-11
- 测试输入：“需求已确认，只帮我生成 Codex 原型提示词。”
- 预期模式：与需求复杂度匹配。
- 预期行为：检查目标、角色、流程、页面、交互和 P0；信息完整时只输出提示词，不额外输出完整 PRD。
- 预期追问数量：信息完整时为 0；存在 P0 时只追问最少必要问题。
- 是否存在 P0：按输入材料。
- 停止追问条件：提示词所需边界完整且无 P0。
- 不应发生的行为：修改代码、新增页面或安装依赖。
- 预期输出：包含实施范围、模拟数据、禁止新增和未覆盖范围的提示词。

## TC-12 直接实现 Demo

- 测试编号：TC-12
- 测试输入：“在当前项目创建这个原型 Demo，使用模拟数据。”
- 预期模式：标准或深度 + Demo 实现。
- 预期行为：先检查 P0、项目结构、新增/改造、模拟数据和文件范围，再实现并验证三类流程。
- 预期追问数量：只补必需实现边界。
- 是否存在 P0：实现前必须为否。
- 停止追问条件：范围、文件、数据和业务逻辑可实现。
- 不应发生的行为：覆盖正式业务文件；修改无关代码；擅自增加规则。
- 预期输出：可运行 Demo、修改清单、运行方式、验证结果和未覆盖项。

## TC-13 用户中途修改需求

- 测试编号：TC-13
- 测试输入：已确认单级审批后，用户改为金额分支多级审批。
- 预期模式：由标准升级为深度。
- 预期行为：覆盖旧决策，重新检查受影响的流程、状态、页面、权限和验收。
- 预期追问数量：1—2 轮处理新 P0。
- 是否存在 P0：是，新阈值和审批链。
- 停止追问条件：新流程闭环且变更影响明确。
- 不应发生的行为：继续使用旧结论；只改页面不改状态和验收。
- 预期输出：变更影响、新状态模型和更新后追踪矩阵。

## TC-14 文档、页面和代码互相冲突

- 测试编号：TC-14
- 测试输入：PRD 说可撤回，页面没有入口，代码在某状态下禁止撤回。
- 预期模式：标准或深度。
- 预期行为：分别记录文档意图、页面表现和代码实现，按优先级处理。
- 预期追问数量：如无最新正式结论，1 个冲突确认问题。
- 是否存在 P0：仅当撤回决定核心状态闭环时是，否则 P1。
- 停止追问条件：冲突采用结论或显式假设明确。
- 不应发生的行为：选最容易实现的代码行为作为 TO-BE。
- 预期输出：信息冲突表、AS-IS/TO-BE/GAP 和变更影响。

## TC-15 AI 功能需求

- 测试编号：TC-15
- 测试输入：“用 AI 自动生成客诉处理建议，客服确认后发送。”
- 预期模式：深度。
- 预期行为：直接将“AI 生成建议、客服确认后发送”提取为已确认，不重复询问人机责任；只补充失败、安全、敏感数据、成本、时延、可解释和兜底等真实缺口。
- 预期追问数量：1—2 轮高相关问题。
- 是否存在 P0：如 AI 是建议还是自动发送未明确，则是；安全与权限风险也可为 P0。
- 停止追问条件：人机责任、数据范围、主流程、失败兜底和验收方向明确。
- 不应发生的行为：默认 AI 可自动对外发送；忽略敏感信息和内容安全。
- 预期输出：人工确认流程、模型异常、安全、成本/时延和效果指标。

## TC-16 已有视频页面增加剪辑包标签

- 测试编号：TC-16
- 测试输入：用户提供已有视频生成页面截图，要求制作人员给视频打标签，剪辑人员将所有已标记视频统一打包下载，并要求不增加制作人员工作流。
- 预期模式：标准。
- 预期行为：优先读取项目 Skill；从截图提取 AS-IS；先识别“新增打标签操作”与“不增加制作人员工作流”的潜在冲突，确认后者是“不新增独立页面/步骤”还是“不增加任何操作”；再处理打包范围和同分镜多版本等会改变闭环的规则，文件命名默认按 P1 合理假设推进；用户逐轮补充时更新确认账本。
- 预期追问数量：只询问会改变主流程或验收的高价值问题，不重复询问截图和对话已明确内容。
- 是否存在 P0：角色、最终交付结果或打包范围完全未知时存在；“不增加工作流”的不同解释会产生两套主流程时也是 P0，解释明确后清除。
- 停止追问条件：制作与剪辑角色、标记入口、打包结果、权限、核心状态、页面范围和验收方向闭环。
- 不应发生的行为：把单次命名确认当作完整需求交付；只输出局部规则；遗漏 AS-IS/TO-BE/GAP、状态、失败恢复、权限差异和验收。
- 预期输出：标准模式正式方案，包括目标与范围、角色权限、主流程与状态、规则、页面和交互、异常、验收、变更影响及文字原型；只有用户明确要求时才附加 Codex 原型提示词。

## TC-17 Skill 读取出现乱码

- 测试编号：TC-17
- 测试输入：在 Windows PowerShell 中读取无 BOM 的 UTF-8 `SKILL.md`，输出出现乱码或不可理解的中文。
- 预期模式：不进入需求分析。
- 预期行为：立即停止任务动作，使用 `Get-Content -Raw -Encoding UTF8` 重新完整读取 Skill 和必要引用文件，确认可读后再执行。
- 是否存在 P0：不适用；这是执行门禁失败。
- 不应发生的行为：依据乱码猜测 Skill 要求；只读取模板文件后继续；声称已经按 Skill 完成。
- 预期输出：重新读取成功后，按项目 Skill 的正常模式完成用户任务。

## TC-18 只要 PRD 不要实现提示词

- 测试编号：TC-18
- 测试输入：“帮我整理成可评审 PRD，不需要原型提示词，也不要改代码。”
- 预期模式：按需求复杂度选择标准或深度。
- 预期行为：完成目标、角色、流程、规则、范围、异常和验收检查，只交付需求方案。
- 预期追问数量：只追问真实 P0。
- 是否存在 P0：按输入材料。
- 不应发生的行为：因为采用标准或深度模式而自动附加 Codex 提示词、实现步骤或代码修改。
- 预期输出：可评审、可测试的正式 PRD。

## TC-19 不同模式的提问上限

- 测试编号：TC-19
- 测试输入：分别给出低风险空状态优化和复杂多角色审批两类模糊需求。
- 预期模式：前者可降级快速，后者深度。
- 预期行为：快速模式每轮最多 3 问；标准/深度通常 1—3 问，只有同一决策无法压缩时最多 5 问。
- 不应发生的行为：使用“最多 3 至 5 问”等模糊口径；快速模式一次提出 4—5 问。
- 预期输出：问题数量、优先级和模式一致。

## TC-20 页面形态不自动升级 P0

- 测试编号：TC-20
- 测试输入：“批量操作用新页面、弹窗还是抽屉我都可以，只要权限和处理结果不变。”
- 预期模式：按主体需求复杂度判断。
- 预期行为：将页面形态标记为 P1/P2 或采用合理假设，不阻塞主流程；只有承载选择连带改变核心结果、权限、状态闭环或已授权实施范围时才升级 P0。
- 是否存在 P0：仅由页面形态产生时不存在。
- 不应发生的行为：因为可能新增页面就自动标记为 P0 并继续追问。
- 预期输出：显式假设和页面选择理由，不影响正式方案交付。
