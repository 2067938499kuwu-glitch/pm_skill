# V2 发布检查清单

## 对话与追问

- [ ] 没有重复追问用户、文档、截图、页面或代码已提供的信息
- [ ] 没有把九步内部框架执行成强制九轮对话
- [ ] 简单需求会合并相关步骤
- [ ] 每轮问题数量和相关性符合当前模式
- [ ] 用户回复“不知道”时已提供低风险可逆选项
- [ ] 用户要求停止时没有继续追问

## P0 与停止条件

- [ ] 没有错误扩大 P0
- [ ] 每个 P0 都符合根本性差异标准之一
- [ ] 默认排序、导出、批量操作、弹窗/抽屉和视觉细节没有被自动标记为 P0
- [ ] 停止追问依据闭环条件，而不是完整度百分比
- [ ] 没有因尚未写完所有 Given/When/Then 而继续追问

## 方案与输出

- [ ] 快速、标准、深度输出已明确区分
- [ ] 没有在简单需求中输出过重模板
- [ ] 已标记已确认、合理假设、待确认、P0 和不在本期
- [ ] 已覆盖核心成功、关键失败和权限差异
- [ ] 已说明实施范围和未覆盖范围
- [ ] 已生成可测试的核心验收标准

## 已有系统与变更

- [ ] 没有把现有代码自动当成目标业务规则
- [ ] 已区分 AS-IS、TO-BE 和 GAP
- [ ] 已标记保留、新增、修改和废弃逻辑
- [ ] 已说明信息冲突来源和采用优先级
- [ ] 已检查页面、角色、权限、状态、数据、接口、口径、迁移、兼容和回滚的适用影响

## 实现边界

- [ ] 未经用户明确要求没有修改代码
- [ ] 没有擅自新增业务功能、页面或依赖
- [ ] 实现 Demo 前已清空 P0 并明确新增/改造范围
- [ ] 实现前已明确模拟数据和文件范围
- [ ] 没有覆盖正式业务文件或修改无关代码

## 非功能与质量

- [ ] 只触发与需求类型匹配的非功能检查
- [ ] 审批/删除/评分/权限已检查日志、审计和追责
- [ ] 数据看板已检查数据源、口径、时间和延迟
- [ ] 外部服务已检查超时、重试、幂等和失败补偿
- [ ] AI 功能已检查失败、安全、成本、响应时间、可解释和人工兜底
