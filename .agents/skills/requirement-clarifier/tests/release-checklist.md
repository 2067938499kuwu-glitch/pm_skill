# V2 发布检查清单

## 加载与执行门禁

- [ ] 项目内存在同名 Skill 时已优先使用项目版本
- [ ] Skill 和必要引用文件均以 UTF-8 正常读取
- [ ] 出现乱码时已停止任务动作并重新读取，没有猜测指令
- [ ] 默认标准模式没有被无理由降级
- [ ] 快速模式每轮最多 3 问；标准/深度通常 1—3 问，仅同一决策无法压缩时最多 5 问
- [ ] 没有把局部确认或零散答复表述为完整需求交付
- [ ] 已识别用户交付意图，没有自动附加未要求的 Codex 提示词或 Demo
- [ ] 需求可能涉及已有系统时，已让用户自主选择关联项目分析或不关联项目分析
- [ ] 用户已明确选择时没有重复询问；未明确选择时没有替用户擅自决定

## 对话与追问

- [ ] 没有重复追问用户、文档、截图、页面或代码已提供的信息
- [ ] 没有把九步内部框架执行成强制九轮对话
- [ ] 简单需求会合并相关步骤
- [ ] 每轮问题数量和相关性符合当前模式
- [ ] 用户回复“不知道”时已提供低风险可逆选项
- [ ] 用户要求停止时没有继续追问

## P0 与停止条件

- [ ] 没有错误扩大 P0
- [ ] 每个 P0 都符合根本性差异标准之一
- [ ] 默认排序、导出、批量操作、弹窗/抽屉和视觉细节没有被自动标记为 P0
- [ ] 停止追问依据闭环条件，而不是完整度百分比
- [ ] 没有因尚未写完所有 Given/When/Then 而继续追问

## 方案与输出

- [ ] 快速、标准、深度输出已明确区分
- [ ] 没有在简单需求中输出过重模板
- [ ] 已标记已确认、合理假设、待确认、P0 和不在本期
- [ ] 已覆盖核心成功、关键失败和权限差异
- [ ] 已说明实施范围和未覆盖范围
- [ ] 已生成可测试的核心验收标准
- [ ] 标准/深度方案中的 G、R、P、I、AC 有可核验的关联

## 已有系统与变更

- [ ] 没有把项目关联方式当作 P0，也没有把“不关联项目”自动判定为全新需求
- [ ] 关联项目分析时已记录可确认的分析基线、分析范围和证据位置
- [ ] 不关联项目分析时没有扫描项目，且已标记 AS-IS 和项目影响未经核验
- [ ] 没有把现有代码自动当成目标业务规则
- [ ] 已区分 AS-IS、TO-BE 和 GAP
- [ ] 已标记保留、新增、修改和废弃逻辑
- [ ] 已说明信息冲突来源和采用优先级
- [ ] 已检查页面、角色、权限、状态、数据、接口、口径、迁移、兼容和回滚的适用影响

## 实现边界

- [ ] 未经用户明确要求没有修改代码
- [ ] 没有擅自新增业务功能、页面或依赖
- [ ] 实现 Demo 前已清空 P0 并明确新增/改造范围
- [ ] 实现前已明确模拟数据和文件范围
- [ ] 没有覆盖正式业务文件或修改无关代码

## 非功能与质量

- [ ] 只触发与需求类型匹配的非功能检查
- [ ] 审批/删除/评分/权限已检查日志、审计和追责
- [ ] 数据看板已检查数据源、口径、时间和延迟
- [ ] 外部服务已检查超时、重试、幂等和失败补偿
- [ ] AI 功能已检查失败、安全、成本、响应时间、可解释和人工兜底
- [ ] 正式需求方案、原型说明和 Demo 均附产品经理能力复盘；阶段性回复和“只要 Codex 提示词”不额外附加
- [ ] 复盘基于本次真实材料、决策和风险；没有空泛表扬、编造遗漏或凑数练习

## 可执行校验

- [ ] 已在 PowerShell 中运行 `scripts/validate-skill.ps1`
- [ ] 已设置 `PYTHONUTF8=1` 并运行 `skill-creator` 的 `quick_validate.py`
- [ ] 严格 UTF-8、替换字符、活动文档链接、测试编号和关键规则一致性全部通过
